# Prayer Memory — 기도 기억 쌓기 & 연결고리 제시

**날짜:** 2026-06-05  
**상태:** 제안

---

## 배경

현재 CLI는 고민 → 말씀 추천 → 구절 출력까지 동작한다.  
`prayer_store.py`가 존재하지만 CLI에 연결되지 않아 기도 이력이 쌓이지 않는다.  
이 기능은 세 번째 핵심 기능("기도: 매일하는 기도를 기억할 수 있도록 한다")을 완성한다.

---

## 목표

1. **기억 stacking** — 기도할 때마다 구조화된 형태로 이력을 저장한다.
2. **연결고리 제시** — 이번에 선택한 말씀과 과거 기도 기록을 비교해, 의미 있는 연결을 화면에 보여준다.

---

## 저장 구조

### 파일: `prayers.json`

현재의 `prayers.md`는 사람이 읽는 로그 역할을 유지한다.  
기계 조회용으로 `prayers.json`을 별도로 운영한다.

```json
[
  {
    "id": "20260605-143022",
    "date": "2026-06-05T14:30:22",
    "concern": "인간관계가 너무 힘들다",
    "episode_id": "오병이어",
    "subtitle": "오병이어",
    "passage_ref": "마 14:13-21"
  }
]
```

| 필드 | 설명 |
|---|---|
| `id` | `YYYYMMDD-HHMMSS` 타임스탬프 |
| `concern` | 사용자가 입력한 고민 원문 |
| `episode_id` | 선택한 에피소드의 subtitle (고유 키) |
| `subtitle` | 표시용 소제목 |
| `passage_ref` | 책 장:절 문자열 |

---

## 저장 시점

구절을 출력한 뒤 한 번 묻는다.

```
[선택한 성경구절]
오병이어 (마 14:13-21)
...

이 말씀으로 기도하셨나요? (y / n)
> y
기도가 기록되었습니다.
```

- `y` → `prayers.json` + `prayers.md` 양쪽에 append
- `n` / 빈 입력 → 저장 생략, 조용히 다음으로

## 연결고리 제시

기도 저장 여부와 관계없이, 구절 출력 직후 아래 두 가지 경우를 순서대로 확인한다.

### Case 1 — 같은 에피소드를 전에도 받은 경우

`prayers.json`에서 `episode_id`가 동일한 레코드를 날짜 역순으로 조회.  
가장 최근 한 건만 표시.

```
[기도 기억]
12일 전 (2026-05-24) — "인간관계가 지쳐서 혼자 있고 싶다"
→ 그때도 이 말씀을 받으셨어요.
```

### Case 2 — 비슷한 고민으로 기도한 기록이 있는 경우

Case 1에 해당하지 않을 때.  
과거 `concern` 텍스트들을 BM25로 인덱스해두고, 현재 고민과 유사도가 가장 높은 레코드를 가져온다.  
유사도가 임계값(예: score > 0)인 경우만 표시.

```
[기도 기억]
3일 전 (2026-06-02) — "아무것도 하기 싫다"
→ 그때는 겟세마네 동산을 받으셨어요.
```

### 기록 없음

두 경우 모두 해당하지 않으면 조용히 생략. 별도 메시지 없음.

## 향후 확장 (메모)

- **아이디어 1 (블렌딩)**: 기도 기록이 충분히 쌓이면 `match()` 점수에 과거 기도 패턴을 가중치로 더하는 방식으로 확장 가능.
- **웹/데스크탑 전환 시**: `prayers.json`은 SQLite 또는 로컬 DB로 교체하기 좋은 구조.
