# BM25 에피소드 매칭 CLI 기획서

**작성일:** 2026-06-05  
**상태:** 확정

---

## 배경 및 결정 컨텍스트

### 기존 MVP 방식의 한계
기존 MVP는 Claude API로 감정/상황 태그를 추출하고 태그 overlap 점수로 구절을 매칭했다.  
Cross-reference를 scrollmapper/bible_databases에서 가져와 적용해봤으나,  
**신학적 해석이 유사한 에피소드를 묶는 경향**이 있어 서사적 병행 관계와 맞지 않았다.

### 이번 기획의 핵심 결정

| 항목 | 결정 |
|------|------|
| 매칭 방식 | BM25 (쿼리 → 에피소드 직접 검색) |
| Translator | 추후 구현 (이번 범위에서 제외) |
| 성경 범위 | 사복음서 (마태/마가/누가/요한, book 40-43) |
| 에피소드 묶음 기준 | **소제목이 같은 경우** 동일 에피소드로 취급 |
| 사용자 제공 방식 | A/B/C 선택지 카드 (최대 3개) |

## 아키텍처

### 데이터 흐름

```
bible.json
    │
    ▼
parse_episodes.py
    │  사복음서 파싱 → 소제목 기준 passage 단위 구조화
    ▼
data/episodes/                ← passage별 JSON 파일 (1-step 요약 포함)
    │
    ▼
build_bm25_index.py           ← BM25 인덱스 구축 (passage 단위)
    │
    ▼
bm25_index/                   ← 인덱스 파일
```

### 에피소드 데이터 구조

```json
{
  "subtitle": "오천 명을 먹이심",
  "books": ["마태복음", "마가복음", "누가복음", "요한복음"],
  "passages": [
    {
      "book": "마태복음",
      "chapter": 14,
      "verse_start": 13,
      "verse_end": 21,
      "verses": [
        {"verse": 13, "text": "..."},
        {"verse": 14, "text": "..."}
      ]
    }
  ],
  "summary": "광야에서 떡 다섯 개와 물고기 두 마리로 오천 명을 먹이심"
}
```

## CLI 플로우

### 전체 흐름

```
사용자 쿼리 입력
        │
        ▼
BM25 Matcher
  - 에피소드 전체 텍스트 대상 검색
  - top 5 에피소드 추출
  - 소제목 동일 에피소드 그룹 병합 (이미 전처리 단계에서 처리됨)
  - 상위 3개 선택
        │
        ▼
A/B/C 카드 출력
  - 소제목
  - 1-step 상황 요약
        │
        ▼
사용자 선택 (A / B / C)
        │
        ▼
에피소드 전체 구절 출력
  - 복음서별 병렬 구절 또는 단일 복음서 구절
```

### 화면 예시

```
오늘 어떤 고민이나 기도제목이 있으신가요?

> 아무리 노력해도 부족한 것 같아요

[관련 말씀을 찾고 있어요...]

A. 오천 명을 먹이심
   적은 것을 내어드렸을 때 예수님이 채우시는 이야기

B. 탕자의 비유
   멀리서 돌아오는 아들을 달려가 맞이하는 아버지의 이야기

C. 포도원 품꾼의 비유
   일한 시간과 무관하게 동일하게 대우받는 일꾼들의 이야기

선택 (A/B/C): A

[선택한 성경구절]
오천 명을 먹이심 (마태복음 14:13-21)
13절  예수께서 이 말씀을 들으시고...
14절  ...

[비슷한 내용의 성경구절]
오천 명을 먹이심 (마가복음 6:30-44)
30절  사도들이 예수께 모여...
31절  ...
```

가장 높은 BM25 score의 passage → "선택한 성경구절"  
동일 소제목의 나머지 복음서 구절 → "비슷한 내용의 성경구절"

## 구현 범위

### 이번에 구현할 것

| 모듈 | 역할 |
|------|------|
| `data/parse_episodes.py` | bible.json → 사복음서 passage 단위 파싱, `data/episodes/` 저장 |
| `data/build_bm25_index.py` | `data/episodes/` → BM25 인덱스 구축 |
| `core/matcher.py` 교체 | BM25 top5 → 소제목 그룹핑 → 상위 3개 반환 |
| `cli/main.py` 수정 | A/B/C 선택 UI, 선택한 구절 + 비슷한 구절 출력 |

### 이번에 구현하지 않는 것

- `core/translator.py` — 구절과 고민을 연결하는 언어 생성 (추후)
- 기도 저장 (`storage/prayer_store.py`) — 추후 재검토

## 데이터 설계 상세

### parse_episodes.py 처리 로직

1. bible.json에서 book 40-43 (마태/마가/누가/요한) 추출
2. 각 구절의 소제목(subtitle) 필드 기준으로 에피소드 그룹핑
3. **소제목이 같은 복음서 간 구절은 하나의 에피소드로 묶음**
4. 출력: `data/gospel_episodes.json`

### build_summaries.py 처리 로직

- 에피소드별로 전체 구절을 Claude Haiku에 전달
- 프롬프트: "이 에피소드의 핵심 상황을 한 문장(20자 이내)으로 요약하세요"
- 결과를 `data/episode_summaries.json`에 저장
- 이미 처리된 에피소드는 건너뜀 (재실행 안전)

### BM25 인덱스 단위

**passage 단위** — 한 복음서 내 소제목 단위가 하나의 문서.  
동일 소제목이라도 마태/마가/누가/요한은 각각 별도 문서로 인덱싱.

```
"예수께서 잡히시다 (마태복음 26:47-56)" → score 10.2
"예수께서 잡히시다 (마가복음 14:43-50)" → score 8.9  ← 사용자에게 노출 안 됨
```

**A/B/C 카드 구성 로직:**
1. BM25 top5 추출 (passage 단위)
2. 소제목 기준 그룹핑 → 소제목당 가장 높은 score의 passage를 대표로 선택
3. 상위 3개 소제목이 A/B/C 카드가 됨

**선택 후 상세 보기:**  
선택된 소제목에 해당하는 **모든 복음서 구절을 함께 출력** (마태 + 마가 등 병렬 표시).
